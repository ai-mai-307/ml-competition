## ML соревнование: IEEE-CIS Fraud Detection — детекция мошеннический транзакций

### Описание задания

Вам предлагается решить практическую задачу **детекции мошеннических онлайн-транзакций**: по набору анонимизированных признаков транзакции и (опционально) признаков устройства/пользователя нужно предсказать **вероятность мошенничества** (`isFraud`). В исходном виде данные состоят из двух частей — **transaction** и **identity**, которые объединяются по ключу `TransactionID`; при этом identity есть не для всех транзакций. 

В Yandex Datasphere вы найдете датасет, где лежит результат объединения этих двух таблиц:

``` python 

train_tr = pd.read_csv(train_tr_path)
train_id = pd.read_csv(train_id_path)

train = train_tr.merge(train_id, on="TransactionID", how="left")
```

Класс `isFraud` сильно несбалансирован, в этом заключается основная сложность данного задания.

---

### Постановка задачи

1. **Получить доступ к данным**: найти датасет ieee-fraud-detection, активировать его и получить доступ к его файлам. Вам доступны обучающее множество данных `train_public.csv`, где есть колонка с таргетом `isFraud` и `test_public.csv`, где колонки с таргетом нет.
2. **Провести предварительный анализ (EDA)**
3. **Сделать sklearn-совместимый пайплайн предобработки**
4. **Реализовать бейзлайн**
5. **Найти и обучить лучшую классическую модель для решения задачи**
6. **Сформировать файл с предсказаниями для данных их `test_public.ipynb`** по образцу `sample_submission.csv`: столбцы `[TransactionID, isFraud]`, где `isFraud` — ваша предсказанная вероятность.
7. **Написать в конце блокнота `solution.ipynb` итоговой описательный отчет о вашем решении задания**: нужно кратко описать итоговый пайплайн, ключевые решения и привести ссылку на таблицу экспериментов.

---

**Метрика соревнования:** **ROC-AUC** между предсказанной вероятностью и истинной меткой (чем больше, тем лучше).
